# 轮盘对决 · 俄罗斯轮盘对战游戏

> 一个支持多道具、多模式、实时对战的回合制游戏。核心逻辑完全自主设计，Web端与部署有AI辅助实现。

## 项目亮点

- **核心游戏引擎**：11种道具、19种游戏模式，支持多模式动态叠加  
- **架构设计**：注册器模式实现道具/模式即插即用；钩子函数链处理复合效果  
- **状态机**：三阶段（选模式/对局/结算）+回合/阶段切换+死亡重赛  
- **完整交付**：QQ机器人+Web对战+Docker部署，已运行于阿里云

## 项目历程

本项目最初是一个与qqbot强耦合的800行的过程式脚本（见`legacy/`目录，其中名为wheel的函数是主入口
，删除了部分敏感信息），实现了基础游戏逻辑。为了提升可维护性和扩展性，我重构为面向对象架构，
并增加了道具/模式的动态注册机制。重构后的核心代码约1200行，新增道具/模式的成本大大降低。
此外，添加了Web端实现和Docker部署。

## 游戏逻辑架构概览

- **Manager**：管理多个群组的游戏实例（每个群独立一个 Handler）
- **CommandHandler**：处理用户指令、数据库持久化、调用 Game 逻辑
- **Game**：核心游戏引擎（状态机、道具、模式等）

## 核心设计思路

### 1. 道具与模式的动态扩展
- 使用 **注册器模式**：新增道具/模式只需继承基类并实现相应方法，框架自动注册  
- 采用 **钩子函数**：模式之间通过修改共享上下文实现叠加

### 2. 状态机管理
- 三阶段流转：选模式 → 对局（回合循环）→ 结算  
- 处理边界：死亡同时发生、重赛、阶段切换时的状态重置  

### 3. 为什么这样设计？
- 关于注册器模式：我需要让道具和模式可以动态扩展。如果不用注册器，每加一个新道具就要改多处判断  
逻辑。注册器把“注册”和“使用”解耦，新增道具只需要写一个类，框架自动识别。这是典型的开闭原则实践。
- 关于钩子函数：多个模式需要影响射击的伤害、目标等。如果用继承，模式之间无法组合。钩子函数让每  
个模式独立修改一个上下文字典，按顺序执行，既支持叠加又不互相污染。
- **关于为什么没继续拆Game类**：我考虑过把回合控制、日志、结算拆成独立类。但当前项目的核心复杂  
度在于道具和模式的规则，而不是流程控制。过度拆分会增加调用链长度，反而降低可读性。所以我选择保
持Game类作为协调中心，属性直接暴露给Mode。

## 快速开始

本项目支持Docker一键启动，无需手动配置环境。

1.克隆项目并进入根目录。  
2.配置环境变量：配置'.env'文件。  
3.启动服务：docker-compose up --build -d  
4.打开浏览器访问http://localhost:8000即可体验。  

## 游戏规则概述

- 道具系统：11种道具（刀、烟、手铐、炸药、灌铅骰子、放大镜等）
- 模式系统：19种模式（西部对决、奇珍异宝、命运之轮等），支持多模式叠加
- bo3对战：阶段1无道具，阶段2每回合1道具，阶段3每回合2道具且低血量无法回血
